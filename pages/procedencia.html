REGISTRO DE PROCEDENCIA DE BAYC

Esta página presenta el registro de procedencia de cada Mono Aburrido que alguna vez existirá. Cada imagen de Bored Ape se procesa primero mediante el algoritmo SHA-256. Se obtiene una cadena combinada concatenando SHA-256 de cada imagen de Bored Ape en el orden específico que se indica a continuación. La prueba final se obtiene mediante el hash SHA-256 de esta cadena combinada. Este es el registro de procedencia final almacenado en el contrato inteligente.

INFORMACION IMPORTANTE

Cada ID de token de Bored Ape se asigna a una imagen de obra de arte de la secuencia inicial con esta fórmula: (tokenId + índice inicial) % 10000 → Índice de secuencia inicial
